Overview

  • C++ is a compiled language

  • Large C++ projects are modular

    • Modules - contains entities to be used by other modules or drivers

    • Drivers - entry point of the program; one driver produces one executable

  • Minimal requirements for C++ development

    • An editor to edit the source code files

    • A building environment to compile the source code files

C++ as a compiled language

Source code files are pure text files containing C++ code

  • header file

    • .hpp files

    • contains contents to be shared by multiple source files

      • shared function declarations

      • shared class declarations

      • shared global variables (not recommended in most cases)

  • source file

    • .cpp files

    • contains contents to be compiled into object files

      • local declarations used only in this file

      • function definitions

      • method definitions

File extensions may vary

Source code files are compiled into object files

  • Object files are binary files containing machine code

    • .o files

    • One .cpp file is compiled into one .o file

  • Object files are linked into executable files

    • Files with executable permission under Linux/Mac OS üëç

    • .exe files under Windows

  • Object files are linked into shared libraries (FYI)

    • .a and .so files under Linux

    • .dylib files under Mac OS

    • .dll files under Windows

Modules

  • Contains entities to be used by other modules or drivers

    • Functions

    • Classes

    • Global variables

  • Files

    • A pair of .hpp and .cpp file with a same base name

    • A single header file

Drivers

  • Entry point of the program

    • a mandatory main() function

  • A project may have multiple drivers

    • Each driver is the entry point of one executable file

    • Application drivers

    • Test drivers

  • A cpp file with a main() function and usually without a paired header

Command-line Interface (CLI)

  • Command-line interface (CLI)

    • A text-based interface to interact with the computer

    • A shell program to interpret the commands

    • A terminal program to display the output

    • Counter part of GUI (Graphical User Interface)

  • bash CLI environment üëç

    • the most popular CLI environment

    • native under Linux and Mac OS

    • available on Windows though WSL, MinGW or CygWin

  • Why CLI?

    • Flexible - adjust parameters on the fly

    • Easy-to-use - easily automated no matter how complicate it is

    • Portable - same commands on different platforms

    • Efficient - less resource consumption

Local Environment

A local development environment is recommended for this course.

  • Something worth learning as a programmer

  • Faster and easier to use than other options

For Linux:

  • Install g++ and make from Ubuntu terminal

    • sudo apt update

    • sudo apt install build-essential

    • sudo apt install git

    • sudo apt install valgrind (optional)

  • Install Visual Studio Code

  • Do research on your own if you are using other Linux distributions

For Windows:

  • Install Ubuntu 20.04 LTS from Microsoft Store

  • Install Visual Studio Code from Microsoft Store

  • Install Terminal from Microsoft Store

  • Start an Ubuntu terminal

  • Install g++ and make in Ubuntu terminal (same as Linux)

Note: To install WSL in the classroom, you must use the Microsoft Store app

For Mac OS:

  • Install XCode Command-line Tools

  • run xcode-select --install in the terminal

  • valgrind not directly available on Mac OS

Traps and Pitfalls ü™§

  • MinGW / MSYS2 / Git Bash

    • A Windows port of g++ toolchain

    • g++ under MinGW is not compatible with g++ under Linux

    • Compatibility issues with make

    • Not recommended for this course

  • Click-and-run single file project

    • Works well for introductory courses

    • No make support

    • Not suitable for modular projects

    • Will fail especially when you are using multiple drivers to generate multiple executables

      • Run and Debug button in VS Code

      • OnlineGDB
