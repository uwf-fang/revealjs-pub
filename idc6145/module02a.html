Reasons for the updates

  • The field of Big Data is evolving fast

  • Terminology is constantly changing

  • New technologies are emerging

  • Updates

    • Modern RDBMS can handle large volumes of data and provide horizontal scalability.

      • e.g. NewSQL

    • Query Engine on Distributed File System

      • Not new but was considered NoSQL in the past

    • Object Store is more commonly referred to as Data Lake now

Data Types of Big Data

  • Structured Data

    • Rigid Schema (Schema-on-write)

    • Queryable with SQL

  • Semi-Structured Data

    • Flexible Schema (Schema-on-read)

    • Queryable with SQL-like language

  • Unstructured Data

    • No Schema

    • Not Queryable

Data Storage Solutions for Big Data

  • Distributed File Systems

    • Query Engine like Apache Hive, Presto, etc. allow for SQL-like queries on large datasets stored in distributed file systems or Data Lakes

  • Data Lake (Commonly as object store)

    • It is more common to refer object store as data lake now

  • NoSQL Databases

  • Modern RDBMS Databases

    • NewSQL Databases

      • Performance and scalability of NoSQL

      • ACID compliance and strong consistency of traditional RDBMS

Choice of Data Storage Solutions by Data Type

  • Structured Data

    • Modern RDBMS

    • NoSQL

    • Query engine on distributed file system or Data Lake

  • Semi-Structured Data

    • NoSQL

    • Query engine on distributed file system or Data Lake

  • Unstructured Data

    • Distributed File System

    • Data Lake (object store)
