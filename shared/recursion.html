Overview

  • Recursion is a technique for solving problems where the solution to a particular problem depends on the solution to a smaller instance of the same problem.

  • Recursive data structures and recursive algorithms.

  • Can replace loops.

Function call mechanism

  • A function call is a request to execute a function.

  • Allocate memory for the functionâ€™s local variables and parameters.

  • Create a new stack frame in the stack memory region.

  • First in last out (FILO) order that is aligned with the stack ADT.

  • Infinite recursion will cause a stack overflow.

Implementation of recursion

  • Recursive function calls itself.

    • Directly: A calls A

    • Indirectly: A calls B, B calls A

  • Base case: A condition that causes the function to return without making recursive calls.

  • Recursive case: A condition that causes the function to make recursive calls.

  • May have more than one base cases.

  • May need forward declaration of the function.

  • May need a helper function to pass additional parameters to keep track of the progress of the recursion.
