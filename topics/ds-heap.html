Overview

  • Definition: A complete binary tree in which parent node is always less/greater (min/max heap) or equal (if allow overlapped values) than all elements in its subtrees.

  • Two type: min and max heap

  • Applications

    • Heap sort

    • Priority queue

Structures

  • Logically a complete binary tree

  • Physically a partially filled array

    • find parent/children by arithmetic calculation of indices

      • given the node \(i\)

      • the index of the parent node is \((i-1)/2\)

      • the indices of the children nodes are left: \(i*2+1\) and right: \(i*2+2\)

Behaviors

  • Heapify: make an array of random numbers into a heap

  • Insert: insert a new element into a heap

    • insert at the end of the array

    • percolate up

  • Delete: delete an element from a heap

    • use the last element to replace the root

    • percolate down

  • Percolate up: move an element up to its proper position

  • Percolate down: move an element down to its proper position

Time Complexity

  • Create a heap

    • From an array (heapify) \(\Theta(n)\)

    • Insert one by one \(\Theta(n\log n)\)

  • Insert \(\Theta(\log n)\)

  • Delete \(\Theta(\log n)\)

  • Percolate up \(\Theta(\log n)\)

  • Percolate down \(\Theta(\log n)\)
