Overview

A hash table is a data structure that maps keys to values for highly efficient lookup. The key-value pairs are stored in “buckets” and the hash table maps keys to buckets using a hash function.

  • Characteristics

    • Fast lookup, insert, and delete \(O(1)\)

    • Unordered

    • Keys are unique

    • Keys are immutable

    • Redundant storage space required

Hash Function For Hash Table

  • \(h(key)\) a special type of hash function used by hash table

  • Maps a key of a certain type to an integer in a certain range \(0, 1, 2, \ldots, size - 1\), where \(size\) is the size of the hash table

  • The value obtained by applying the hash function to a key is called the hash value of the key. It is used as an index of the bucket.

  • Modulo arithmetic

  • Considerations

    • Deterministic

    • Fast to compute

    • Uniform distribution (reduce clustering and collision)

  • Collision

    • More than one keys map to the same index

Collision Resolution

  • The most important component of a hash table!

  • Optimal time complexity \(O(1)\)

  • Load factor \(\alpha = \frac{n}{size}\)

    • where \(n\) is the number of key-value pairs

    • \(size\) is the size of the hash table

    • Load factor is a measure of how full the hash table is

    • \(\alpha\) should be ideally a small constant (i.e. 0.5) to ensure \(O(1)\) time complexity for lookup, insert, and delete

  • Two approaches

    • Separate chaining (or simply chaining)

    • Open addressing

Chaining

  • Each bucket can store a variable number of key-value pairs

    • \(O(nk)\) space complexity

      • where \(n\) is the number of buckets

      • \(k\) is the average number of key-value pairs in each bucket

      • Theatrically unlimited space for each bucket

      • \(k\) should be ideally a small constant

    • \(O(k)\) time complexity for lookup, insert, and delete

  • Solve collision by chaining the key-value pairs together in one bucket when collision happens

Open Addressing

  • Each bucket stores at most one key-value pair

    • \(O(n)\) space complexity

      • Can store at most \(n\) key-value pairs

    • \(O(k)\) time complexity for lookup, insert, and delete

      • \(k\) is the average number of probing steps

      • \(k\) should be ideally a small constant

  • Solve collision by finding another available bucket using a “probing sequence”

    • Local array

    • Dynamic array

    • Vector

  • Bucket

    • Stores a key-value pair

    • Stores empty status

      • Empty since start

      • Empty after deletion

};

Probing in Open Addressing

  • find another available bucket to store the value when a collision occurs

  • probing sequence: the sequence of the buckets to check when collision happens

  • linear probing:

    \((h1(key) + C_{1} * i) \% NUM\_BUCKETS, i = 0, 1, 2, 3 \dots\)

  • quadratic probing:

    \((h1(key) + C_{1} * i + C_{2} * i^2) \% NUM\_BUCKETS, i = 0, 1, 2, 3 \dots\)

  • double hashing probing:

    \((h1(key) + i * h2(key)) \% NUM\_BUCKETS, i = 0, 1, 2, 3 \dots\)

  • \(h1(key)\) is the primary hash function

  • \(h2(key)\) is the secondary hash function

  • \(C_{1}\) and \(C_{2}\), etc. are constants

Applications of Hash Table

  • Implement Map ADT (Hash Map)

    • A.k.a dictionary, associative array, symbol table, etc.

    • A collection of key-value pairs

    • Keys are unique

    • Behaviors

      • Set a key-value pair

      • Get the value of a key (lookup, contains)

      • Delete a key-value pair

  • Implement Set ADT (Hash Set)

    • A collection of unique elements

    • No value stored

    • Only store keys (or say key is the value)

    • Keys are unique

    • Behaviors

      • Add a key

      • Remove a key

      • Check if a key exists

      • Set operations: Union, intersection, difference, subset, etc.
