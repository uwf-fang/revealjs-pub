Overview

  • Broad definition: Hashing is a process of mapping data of one type to data of another type.

  • Narrow definition: Hashing is a process of mapping data of arbitrary size to data of fixed size.

  • Hash function

    • The core of hashing

  • Representation: <start type> -> <destination type>

    • <start type>: The type of data to be hashed, sometimes called key type

    • <destination type>: The type of data after hashing, sometimes called hash value type or hash code type

    • E.g. int -> int, int -> string, string -> int, string -> string

Hash Function

  • Domain of a hash function: The set of all possible inputs to a hash function

  • Range of a hash function: The set of all possible outputs of a hash function

  • Characteristics (not all required):

    • Deterministic

    • Constant time computation \(O(1)\)

    • Uniform distribution (clusters in domain are evenly distributed in range)

    • Size of range is smaller usually than size of domain

    • Asymmetric

      • Hard/impossible to reverse

      • Required in certain applications like cryptography

    • Chance of collision

Design of Hash Functions

  • Integer keys

  • String keys

    • Handle variable length

    • All characters are involved

    • Uniform distribution

  • Asymmetric hash functions

    • Hard to reverse

    • Cryptographic hash functions

    • E.g. MD5, SHA-1, SHA-2, SHA-3
