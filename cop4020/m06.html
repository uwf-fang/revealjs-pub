Introduction to Data Types

  • Definition: Collection of data values and predefined operations

  • Object-oriented perspective

    • Primitive data types and objects: C++, Java, C#, D, Delphi, Ada, Go, Swift

    • Everything is an object: Python, Ruby, Javascript, Smalltalk

Primitive Data Types

  • Definition: Data types that are not objects

  • Advantage

    • Efficiency: Primitive data types are faster than objects

    • Simplicity: Primitive data types are easier to implement

  • C

    • base: int, short, long, float, double, char, bool

    • derived: array, pointer, struct, union, enum

All Objects

  • Definition: All data types are objects

  • Advantage

    • Flexibility: All data types can be extended

    • Consistency: All data types have the same syntax

  • Python

    • everything derived from object

    • type is an object too

    • all operations are method calls

    • everything is extensible

Representative Data Types

  • Numeric: int, float, complex, decimal

  • Boolean

  • Character: char, string

  • Enumeration

  • Array, List, Tuple

  • Set, Dictionary

  • Memory address: Pointers and References

  • Composite: struct, union, class

Language Support for Data Types

  • Type checking

    • Static: C, C++, Java, C#, D, Swift

    • Dynamic: Python, Ruby, Javascript

  • Strong typing: type error is always detected

  • Type equivalence

    • Name equivalence: C, C++, Java, C#, D, Swift

    • Structure equivalence: Pascal, Ada, Go

  • Type system

Language Implementation

  • C/C++

    • No run-time support

    • automatic stack

    • manual heap

  • Java

    • JVM: Java Virtual Machine

    • automatic stack and heap

    • garbage collection

  • Python

    • CPython, Jython, IronPython, PyPy implementations behaves differently

    • automatic stack and heap

    • garbage collection

Expressions and Assignment

  • Expression specifies computation

  • Syntax usually as a context-free language

  • Concepts

    • Operands: variable, literal

    • Operators: unary, binary, ternary

    • Precedence, associativity

    • type matching

    • Coercing

    • Short-circuit evaluation

    • Side-effect of assignment

      • essential in imperative PL

      • avoid in functional PL

Arithmetic Expressions

  • Types

    • unary, binary and ternary

    • prefix, infix and postfix

  • Order of evaluation

    • Precedence

    • Associativity

    • Parentheses

  • LISP expression

    • Operator as function name (+ a (* b c))

  • Conditional expression

    • C (a > b) ? a : b

    • Python a if a > b else b

Prefix Infix Postfix

  • Prefix

    • Polish notation

    • Example: + 3 4

  • Infix

    • Traditional notation

    • Example: 3 + 4

  • Postfix

    • Reverse Polish notation

    • Example: 3 4 +

  • Prefix and Postfix

    • No parentheses

    • Easy to parse

    • Hard to read

  • Infix

    • Parentheses

    • Easy to read

    • Hard to parse

Side-effects in Expressions

  • Part of the expression changed the value of another part of the expression

  • Example

    result = (a++ + b) / (a-- - c)

if ((a = 10) > 0) {
  a++;
}

  • Referential transparency

    • Pure function

    • No side-effect

Type conversion

  • Widening conversion

    • Implicit conversion

    • No loss of information

  • Coercion (implicit type conversion)

    • in javascript

      "7" + 5 = "75"
"7" == 7
"7" === 7

  • Explicit type conversion

    • C++ static_cast<int>(3.14)

    • Python int(3.14)

    • JavaScript parseInt("3.14")
