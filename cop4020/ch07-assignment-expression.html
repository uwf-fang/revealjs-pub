Expressions and Assignment

  • Expression specifies computation

  • Syntax usually as a context-free language

  • Concepts

    • Operands: variable, literal

    • Operators: unary, binary, ternary

    • Precedence, associativity

    • type matching

    • Coercing

    • Short-circuit evaluation

    • Side-effect of assignment

      • essential in imperative PL

      • avoid in functional PL

Arithmetic Expressions

  • Types

    • unary, binary and ternary

    • prefix, infix and postfix

  • Order of evaluation

    • Precedence

    • Associativity

    • Parentheses

  • LISP expression

    • Operator as function name (+ a (* b c))

  • Conditional expression

    • C (a > b) ? a : b

    • Python a if a > b else b

Prefix Infix Postfix

  • Prefix

    • Polish notation

    • Example: + 3 4

  • Infix

    • Traditional notation

    • Example: 3 + 4

  • Postfix

    • Reverse Polish notation

    • Example: 3 4 +

  • Prefix and Postfix

    • No parentheses

    • Easy to parse

    • Hard to read

  • Infix

    • Parentheses

    • Easy to read

    • Hard to parse

Type conversion

  • Widening conversion

    • Implicit conversion

    • No loss of information

  • Coercion (implicit type conversion)

    • in javascript:

      "7" + 5 = "75"
"7" == 7
"7" === 7

  • Explicit type conversion

    • C++ static_cast<int>(3.14)

    • Python int(3.14)

    • JavaScript parseInt("3.14")
