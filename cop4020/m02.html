Overall simplicity

Orthogonality

  • Ability of combining a small set of primitives to build control and data structures.

  • The more orthogonal the design of a language, more is the simplicity.

  • Most orthogonal programming language is ALGOL 68.

  • Functional languages offer potentially the greatest overall simplicity

Support for Abstraction

  • Abstraction is the ability to define and use complicated structures or operations in ways that allow many of the details to be ignored.

  • Programming languages can support two types of abstraction: process and data.

  • Example for process abstraction is to use a subprogram for sorting that is required several times in a program.

  • Example for data abstraction is a binary tree that stores integer data in its nodes. It is better to implement a binary tree in C++ or Java using classes rather than in Fortran77 using pointers and dynamic memory management.

Expressivity

  • Ability of a language that allows a great deal of computation to be accomplished with a very small program.

  • In C, count++ is more convenient than writing count = count + 1.

  • Inclusion of for statement in Java makes writing counting loops easier than with the use of while.

Formal Languages

  • Mathematical system to describe syntax and semantics of languages.

    • Alphabet: finite set of symbols (Σ)

    • String: finite sequence of symbols from alphabet

    • Language: set of strings following specific rules

    • Grammar: rules for generating valid strings

In Lexical Analysis:

  • Alphabet: ASCII character

  • String: Lexeme

  • Grammar: Rule to construct lexemes

  • Tokens: Category of lexemes

In Syntax Analysis:

  • Alphabet: Token

  • String: Program

  • Grammar: Syntax rules

Symbolic Notation in Formal Languages

  • Terminal (a, b, …): single character in the alphabet, cannot be expanded

  • Nonterminal (A, B, …): can be expanded

  • Terminal or nonterminal (X, Y, …):

  • Strings of terminal (w, x, …): sequence of terminal symbols

  • Mixed strings \(\alpha, \beta, \gamma, ...\): sequence of terminal and nonterminal symbols

Note

BNF (Backus-Naur Form) and its variations are a widely used notation to describe formal grammars in the context of programming languages and compilers.

Automata Types

  • Finite automata: Deterministic and non-deterministic Finite Automata (DFA, NFA)

  • Pushdown automata: stack memory

  • Linear-bounded automata: limited tape memory

  • Turing machines: infinite tape memory

Introduction to Syntax and Semantics

  • Syntax: form or structure of the expressions, statements, and program units

  • Semantics: meaning of the expressions, statements, and program units

  • Concepts:

    • Lexemes: lowest-level syntactic units in a language

    • Tokens: a category of lexemes

  • Components:

    • Language recognizer: syntax analyzer/parser

    • Language generator

Formal Methods of Describing Syntax

  • Backus- Naur Form (BNF)

    • a metalanguage used to formally describe the syntax of a language

    • for context-free grammars

  • Extended BNF (EBNF)

  • Grammar

    • a generative device for defining languages

Example Grammar in BNF

<program> → begin <statement list> end
<statement list> → <statement>
                | <statement>; <statement list>
<statement> → <var> = <expression>
<var> → A | B | C
<expression> → <var> + <var>
            | <var> - <var>
            | <var>

Derivation Example

Prove that “begin A = B + C; B = C end” is a program in the grammar.

<program> → begin <statement list> end
          → begin <statement>; <statement list> end
          → begin <var> = <expression>; <statement list> end
          → begin A = <expression>; <statement list> end
          → begin A = <var> + <var>; <statement list> end
          → begin A = B + <var>; <statement list> end
          → begin A = B + C; <statement list> end
          → begin A = B +; <statement> end
          → begin A = B + C; <var> = <expression> end
          → begin A = B + C; B = <expression> end
          → begin A = B + C; B = <var> end
          → begin A = B + C; B = C end

Attribute Grammars

  • Support certain context-sensitive semantics by associating attributes with grammar symbols

  • Parser extension

    • Add attributes to tokens and nonterminals

    • Add semantic rules to the grammar

  • Type of attributes

    • Synthesized attributes - bottom-up attributes that are computed from the attributes of the children

    • Inherited attributes - top-down attributes that are passed from the parent to the children

Meaning of Programs

  • Static semantics: meanings that can be determined at compile time

  • Dynamic semantics: meanings that can be determined at run time, referred as semantics in the rest of this course!

  • Operational semantics: meanings of programs are defined by interpreting their execution on a machine

    • natural operational semantics

    • structural operational semantics

  • Denotational semantics: meanings of programs are defined by mathematical objects that represent the effect of executing the programs

    • syntactic domain

    • semantic domain
