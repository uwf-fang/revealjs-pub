Cluster Managers for Spark

  • Cluster manager: software that manages resources across a cluster

  • Major cluster managers for Spark:

    • Standalone

    • YARN

    • Mesos

    • Kubernetes (containerized workloads)

    • AWS EMR (Elastic MapReduce)

Spark on Cluster

  • Spark can run in local mode without a cluster manager (for development)

  • Deploy Modes:

    • Cluster mode: driver runs on a worker node

    • Client mode: driver runs on the client machine

  • General process:

    • Start cluster manager on every node

    • Submit application to cluster manager

      • spark-submit or spark-shell

      • Specify master URL

    • Cluster manager launches executors on worker nodes

    • Executors run tasks for the application

Standalone Cluster Manager

  • Simple cluster manager that comes with Spark (no additional installation)

  • Architecture

    • Master process: can run on any node

    • Worker processes

  • Scripts

    • start-master.sh

    • start-slave.sh

    • stop-master.sh

    • stop-slave.sh

    • start-all.sh

    • stop-all.sh

Apache Mesos Cluster Manager

  • General-purpose cluster manager

    • Scalability to 10,000s of nodes

    • Fault-tolerant masters and slaves

    • Multi-resource (CPU, memory, disk, and ports) scheduling

    • Support for Docker containers

    • Native isolation between tasks

  • Architecture

    • Master: 1 with optional standby, runs Spark scheduler

    • Slave: many, runs Spark executors

  • Running mode:

    • Fine-grained mode: dynamic sharing of resources

    • Coarse-grained mode: single long-running task per slave for each spark application
