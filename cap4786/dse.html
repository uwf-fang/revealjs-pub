Concepts

  • Compute: the cluster that runs your code

  • Workspace: the place to store your notebooks, libraries, and files

  • Notebook: the place to write and run your code and report/documents

  • Relationship:

    • A workspace can have multiple users

    • A user has multiple notebooks

    • A notebook must be attached to a cluster to run code

Notebook interface

  • Notebook

    • A place to run code

    • A report with code, run results, and text

    • Export to source code to keep the code

    • Export to HTML to share the report (used for assignment)

  • Markdown cell: write text, equations, images, links, etc.

    • Learn Markdown syntax here

  • Code cell: write and run code

  • Language support: Python, Scala, SQL (Spark SQL), R

    • top left corner: select language for the notebook

    • in code cell: specify language for the cell using %python, %scala, %sql, %r

  • Run and compute control: top right

    • manage a compute in the dropdown menu

  • Other important feature

    • quick access to Spark UI and other compute management interfaces

Compute Management Interface

  • Term “compute” refers to the cluster that runs your code

  • Detailed compute management

    • Create

    • Terminate

    • Clone

    • Delete

    • You cannot resume a terminated compute

  • Report and dashboard view

    • compute status

    • Spark UI

  • Access Web terminal

    • Useful for running shell commands

    • Databricks features not available in the terminal

Workflow

  • Always start with creating a compute

    • It is very slow!!!

    • Terminate automatically after 1-2 hours of inactivity

  • Create a notebook

  • Obtain data

    • From URL

    • From Databricks datasets

    • Upload your own

  • Write code

  • Make a report

  • Export to HTML

    • It is ready for assignment submission

    • Do not use the display() function! It will make the HTML file huge!

  • Download the data (optional)

    • Copy/Move data file to dbfs:/FileStore/ directory

    • Use REST API to download the data
