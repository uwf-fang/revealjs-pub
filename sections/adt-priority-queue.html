Overview

  • A priority queue ADT is a container of elements that guarantees that the element with the highest priority is the first element to be obtained and removed from the container.

  • It is not really a queue because it is not FIFO!

  • Characteristics

    • Homogeneous

    • Access order is highest/lowest rank first

    • Finite

    • Dynamic size

  • Applications

    • Many greedy algorithms

    • Event-driven simulation

    • Dijstraâ€™s shortest path algorithm

Behaviors

  • enqueue: Add an element to the priority queue.

  • dequeue: Remove and return the value of the element with the highest/lowest rank from the priority queue.

  • peek: Return the element to be removed from the queue.

  • isEmpty: Return true if the stack is empty, false otherwise.

  • size (optional): Return the number of elements in the container.
