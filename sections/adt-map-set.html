Map and Set ADTs

  • Overview

  • Behaviors

Overview

  • A map ADT is a collection of key-value pairs, where each key is unique and is used to retrieve the corresponding value.

  • A set ADT is a collection of unique elements, with no specific order.

  • Characteristics

    • Homogeneous

    • Keys in map are unique

    • Elements in set are unique

    • Finite

    • Dynamic size

  • Applications

    • Database indexing (map)

    • Caching (map)

    • Key-value storage (map)

    • Membership testing (set)

    • Removing duplicates from a collection (set)

Behaviors

Map Behaviors

  • put: Insert a key-value pair into the map.

  • get: Retrieve the value associated with a given key.

  • remove: Remove a key-value pair from the map using the key.

  • containsKey: Check if a key exists in the map.

Set Behaviors

  • add: Insert an element into the set.

  • remove: Remove an element from the set.

  • contains: Check if an element exists in the set.
