Queue Abstract Data Type

  • Overview

  • Behaviors

Overview

  • A queue ADT is a container of elements that are inserted and removed according to the first-in-first-out (FIFO) principle.

  • A.k.a. FIFO queue, waiting line, pipe, etc.

  • Characteristics

    • Homogeneous

    • Access order is FIFO/LILO

    • Finite

    • Dynamic size

  • Applications

    • Breadth-first search

    • Scheduling

    • Buffering

  • Known as one of the linear ADT because it is usually implemented using a linear data structure. It is not logically linear.

Behaviors

  • enqueue: Add an element to the queue.

  • dequeue: Remove and return the value of the element with the highest/lowest rank from the queue.

  • peek: Return the element to be removed from the queue.

  • isEmpty: Return true if the stack is empty, false otherwise.

  • size (optional): Return the number of elements in the container.
